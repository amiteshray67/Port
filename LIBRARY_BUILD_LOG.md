## Architecture

### Canonical Source of Truth: Topic Graph
Everything maps into one ground-truth CFA L2 topic tree.
- NOT pairwise links between books (that's N² complexity)
- N sources × 1 canonical tree = N mappings
- New book? Just map it to the tree. Cross-refs auto-derive.

### Three Layers
1. **Canonical Topic Taxonomy** — `library/data/topic-graph.json`  
   `Quant > Multiple Regression > Heteroskedasticity > Breusch-Pagan Test`
2. **Per-Source Index** — `library/data/{bookId}.idx.json`  
   `[{topic_id, page_start, page_end, heading, confidence}]`
3. **Cross-Reference Resolver** — runs in-browser at read time  
   Page X of Schweser → lookup canonical topics → find same topics in Institute volumes → show links in right pane

### Hosting Strategy
- **HTML/JS/JSON** → GitHub → CI deploy to Hostinger (same pattern as studyhub)
- **PDFs** → FTP direct to Hostinger `/domains/amiteshray.com/public_html/library/pdfs/`  
  (excluded from git via .gitignore — they're binary, large, and copyrighted)
- **No R2 needed**: Institute volumes are 2–7MB each. Notes PDFs are 47–75MB but still FTP-able.
- **Password gate**: JS gate using SHA-256 hash. Not bulletproof but prevents casual access to copyrighted Institute curriculum.

**Script:** `library/scripts/indexer.py`

**Algorithm:**
1. For each PDF: extract TOC (via PyMuPDF bookmarks) + headings (font-size heuristics)
2. Map headings → canonical topic graph (fuzzy match first, embeddings for rest)
3. Output: `{bookId}.idx.json` per PDF

## Phase 3 — Cross-Reference UI + AI Q&A + Mock Linkbacks (TODO)

**Cross-ref resolver:**
```javascript
function getPageTopics(bookId, pageNum) {
  const idx = loadedIndexes[bookId];
  return idx.filter(entry => pageNum >= entry.page_start && pageNum <= entry.page_end);
}
function getCrossRefs(bookId, pageNum) {
  const topics = getPageTopics(bookId, pageNum);
  const refs = [];
  for (const topic of topics) {
    for (const [otherId, otherIdx] of Object.entries(loadedIndexes)) {
      if (otherId === bookId) continue;
      const matches = otherIdx.filter(e => e.topic_id === topic.topic_id);
      refs.push(...matches.map(m => ({...m, bookId: otherId})));
    }
  }
  return refs;
}
```

## Key Technical Decisions

| Decision | Choice | Reason |
|---|---|---|
| PDF renderer | PDF.js 3.11.174 (CDN legacy build) | Battle-tested, no build step, works in single HTML file |
| State storage | IndexedDB via `idb-keyval` pattern | Handles large highlight bodies, faster than localStorage |
| Text layer | Manual span positioning | Full control, no dependency on renderTextLayer API changes |
| Highlight positions | Text + page number (no pixel coords in Phase 1) | Simpler, still useful; pixel coords in Phase 2 |
| SRS algorithm | SM-2 (simplified) | Industry standard, simple to implement, very effective |
| Auth | SHA-256 gate, password `cfa2026` | Personal tool; prevents casual access to copyrighted PDFs |
| PDF hosting | FTP direct to Hostinger `/library/pdfs/` | PDFs excluded from git (too large + copyrighted) |
| Cross-ref approach | Canonical topic graph (not pairwise) | N mappings not N² |
| Embedding model | Phase 2: all-MiniLM-L6-v2 (runs locally via sentence-transformers) | Free, fast, good for domain text |
